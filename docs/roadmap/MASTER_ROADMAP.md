# MASTER_ROADMAP

## Overview
This roadmap outlines the development plan for the RESTI POS system over the next 8 weeks, based on the consolidated audit reports.

## Sprint 0 – Hotfix Sprint (Week 1)
- Fix critical blocking issues across backend, frontend, and DevOps.
- Standardize API endpoints.
- Resolve database inconsistencies and incorrect ORM mappings.
- Stabilize KDS state machine and ensure proper event flows.
- Fix Docker port mismatches and reduce security risks.

## Sprint 1 – System Integration (Weeks 2-3)
- Build unified UI component library.
- Implement consistent error handling and toast notifications.
- Migrate KDS to WebSocket-based real-time updates.
- Begin database normalization and migration planning.
- Implement search debounce and optimize API load.

## Sprint 2 – Major Refactor (Weeks 4-6)
- Full database migration with improved relational integrity.
- Refactor microservice communication links.
- Finish logistics and courier dispatch system.
- Implement drag-and-drop table layout with room grouping.
- Introduce new VIP module workflows.

## Sprint 3 – Feature Implementation (Weeks 7-8)
- Enhanced kitchen priority workflow.
- Full payment module with split payments and receipt logic.
- Improve delivery prediction with zone-based ETA adjustments.
- Implement loyalty and customer profile system.
- Finalize UI visual redesign for production.

## Deliverables
- Stable, production-ready backend and frontend.
- Fully documented system architecture.
- Comprehensive test coverage across services.
- Unified UI/UX design across all modules.
